Introduction

JavaScript est un langage de script multiplateforme orienté objet. Il est un langage petit et léger. À l'intérieur d'un environnement hôte (pour par exemple, un navigateur Web), JavaScript peut être connecté aux objets suivants son environnement pour en assurer le contrôle programmatique.

JavaScript contient une bibliothèque standard d'objets, tels que Array, Date, et les mathématiques, ainsi qu'un ensemble d'éléments langagiers de base tels que les opérateurs, les contrôles des structures et des déclarations. JavaScript de base peut être étendu à un fichier en la complétant par des objets supplémentaires ; pour exemple :

  • JavaScript côté client étend le langage de base en fournissant des objets pour contrôler un navigateur et son Document Object Model (DOM). Par exemple, permettent à une application de placer des éléments sur un fichier Formulaire HTML et de répondre aux événements utilisateur tels que les clics de souris, la saisie de formulaire, et la navigation par page.
  • JavaScript côté serveur étend le langage de base en fournissant des objets concernant l'exécution de JavaScript sur un serveur. Par exemple, côté serveur permettent à une application de communiquer avec une base de données, assurer la continuité de l'information d'une invocation à l'autre de l'application, ou effectuer des manipulations de fichiers sur un serveur.

JavaScript et Java

JavaScript et Java sont similaires à certains égards, mais fondamentalement différents. dans d'autres. Le langage JavaScript ressemble à Java mais n'a pas de Saisie statique de Java et vérification de type forte. JavaScript suit la plupart Syntaxe des expressions Java, conventions de nommage et flux de contrôle de base ce qui explique pourquoi il a été renommé de LiveScript à JavaScript.

Contrairement au système de compilation de classes de Java construit par JavaScript prend en charge un système d'exécution basé sur un petit fichier nombre de types de données représentant des valeurs numériques, booléennes et des chaînes de caractères. JavaScript a un modèle objet basé sur un prototype au lieu du modèle objet le plus courant. modèle objet basé sur la classe. Le modèle basé sur un prototype permet d'obtenir des données dynamiques l'héritage, c'est-à-dire que ce qui est hérité peut varier d'un objet à l'autre. JavaScript prend également en charge les fonctions sans aucune déclarativité particulière. exigences. Les fonctions peuvent être des propriétés d'objets, exécutées comme des méthodes dactylographiées de façon lâche.

JavaScript est un langage de forme très libre comparé à Java. Vous ne devez pas doivent déclarer toutes les variables, classes et méthodes. Vous n'avez pas besoin de se préoccuper de savoir si les méthodes sont publiques, privées ou protégées, et vous n'avez pas besoin d'implémenter des interfaces. Variables, paramètres et les types de retour de fonction ne sont pas explicitement saisis.

Variables

Vous utilisez les variables comme noms symboliques des valeurs dans votre application. Le les noms de variables, appelés identificateurs, sont conformes à certaines règles.

Un identificateur JavaScript doit commencer par une lettre, un trait de soulignement (_), ou ($) ; les caractères suivants peuvent aussi être des chiffres (0-9). Parce que JavaScript est sensible à la casse, les lettres comprennent les caractères "A" jusqu'à "Z " (majuscules) et les caractères "a" à "z" (minuscules).

Vous pouvez utiliser les lettres ISO 8859-1 ou Unicode telles que å et ü dans les identificateurs. Vous pouvez également utiliser les séquences d'échappement Unicode comme caractères dans des identificateurs. Quelques exemples de noms légaux sont Number_hits, temp99, et _nom.

Déclaration de variables
Vous pouvez déclarer une variable de trois façons :

Avec le mot-clé var. par exemple, var x = 42. Cette syntaxe peut être utilisé pour déclarer des variables locales et globales.

En lui attribuant simplement une valeur. Par exemple, x = 42. Ce déclare toujours une variable globale. Il génère un JavaScript strict avertissement. Vous ne devriez pas utiliser cette variante.

Avec le mot clé let. Par exemple, let y = 13. Cette syntaxe peut être utilisée pour déclarer une variable locale de portée de bloc. Voir Périmètre variable ci-dessous.

Portée de variable

Quand vous déclarez une variable en dehors de n'importe quelle fonction, elle s'appelle une variable parce qu'elle est disponible pour n'importe quel autre code de la variable globale document actuel. Lorsque vous déclarez une variable à l'intérieur d'une fonction, c'est appelée variable locale, parce qu'elle n'est disponible qu'à l'intérieur de cette variable. fonction.

JavaScript avant ECMAScript 2015 n'a pas de portée d'instruction de bloc ; plutôt, une variable déclarée dans un bloc est locale à la fonction (ou ) à l'intérieur duquel le bloc réside. Par exemple, les éléments suivants enregistrera 5, parce que la portée de x est la fonction (ou global contexte) dans lequel x est déclaré, pas le bloc, qui dans ce cas-ci est une déclaration "si".

if (true) { var x = 5; } console.log(x); // 5

Ce comportement change lorsque l'on utilise la déclaration let introduite dans ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Variables globales

Les variables globales sont en fait des propriétés de l'objet global. Dans le web l'objet global est une fenêtre, ce qui vous permet de définir et d'accéder aux pages globales. en utilisant la syntaxe window.variable.

Par conséquent, vous pouvez accéder aux variables globales déclarées dans une fenêtre ou à partir d'une autre fenêtre ou d'un autre cadre en spécifiant la fenêtre ou le cadre nom. Par exemple, si une variable appelée phoneNumber est déclarée dans un fichier vous pouvez faire référence à cette variable à partir d'un iframe sous la forme parent.phoneNumber.

Déclarations if et else
Utilisez l'instruction if pour exécuter une instruction si une condition logique est vrai. Utilisez la clause optionnelle pour exécuter une proposition si la condition est fausse. Une déclaration de si ressemble à ce qui suit : if (condition) { statement_1 ; } else { statement_2 ; } condition peut être n'importe quelle expression qui évalue à vrai ou faux. Voir Booléen pour une explication de ce qui est évalué comme vrai et faux. Si est évaluée à vrai, l'instruction_1 est exécutée ; sinon, statement_2 est exécuté. statement_1 et statement_2 peuvent être n'importe quelle instruction, y compris d'autres déclarations imbriquées.

Vous pouvez également composer les énoncés en utilisant d'autres moyens si vous voulez avoir plus d'un énoncé. conditions testées dans l'ordre, comme suit :

if (condition_1) { statement_1 ; } sinon if (condition_2) { statement_2 ; else if (condition_n) { statement_n ; } else { statement_last ; } Dans le cas de conditions multiples, seule la première condition logique qui évalue à true sera exécuté. Pour exécuter plusieurs instructions, regroupez à l'intérieur d'une instruction de bloc ({ .... }) . En général, c'est une bonne pratique d'utiliser toujours les instructions de bloc, en particulier lors de l'imbrication des instructions if : if (condition) { statement_1_runs_if_if_condition_is_true ; statement_2_runs_if_condition_condition_is_true ; } else { statement_3_runs_if_condition_condition_is_false ; statement_4_runs_if_condition_condition_is_false ; } Il est conseillé de ne pas utiliser d'assignations simples dans une expression conditionnelle, parce que la mission peut être confondue avec l'égalité quand on jette un coup d'œil par-dessus la tête le code. Par exemple, n'utilisez pas le code suivant : if (x = y) { /* instructions ici */ } Si vous avez besoin d'utiliser un code dans une expression conditionnelle, une pratique courante consiste à mettre l'expression des parenthèses supplémentaires autour de l'affectation. Par exemple : if ((x = y)) { /* instructions ici */ }
Déclaration While
Une instruction while exécute ses instructions aussi longtemps qu'une condition spécifiée s'évalue à vrai. Une déclaration de temps ressemble à ce qui suit : while (condition) statement If the condition becomes false, à l'intérieur de la boucle arrête l'exécution et contrôle les passes vers l'instruction en suivant la boucle.

Le test de condition se produit avant l'exécution de l'instruction dans la boucle. Si la condition retourne true, l'instruction est exécutée et la condition est encore une fois testé. Si la condition renvoie false, l'exécution s'arrête et est passé à l'instruction qui suit while.

Pour exécuter plusieurs instructions, utilisez une instruction de bloc ({ .... }) pour grouper ces déclarations.

Exemple :

Les éléments suivants pendant l'itération de boucle tant que n est inférieur à trois:

var n = 0 ; var x = 0 ; while (n < ; 3) { n+++ ; x += n ; }.

A chaque itération, la boucle incrémente n et ajoute cette valeur à x. Par conséquent, x et n prennent les valeurs suivantes :

  • Après la première passe : n = 1 et x = 1
  • Après la deuxième passe : n = 2 et x = 3
  • Après la troisième passe : n = 3 et x = 6

Après le troisième passage, la condition n < ; 3 n'est plus valide. true, donc la boucle se termine.
